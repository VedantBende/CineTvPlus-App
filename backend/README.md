# ⚙️ CineTv+ Backend

This directory comprises the Node.js/Express server that powers the core logic, security boundaries, and database operations for the **CineTv+** platform.

---

## 🛠️ Tech Stack & Key Libraries

- **Runtime Environment**: Node.js
- **Web Framework**: **Express.js** for robust API routing and middleware management.
- **Database**: **MongoDB** (Atlas) managed via the **Mongoose** ODM for strict schema enforcement.
- **Security**: 
  - **Helmet**: Secures HTTP headers and enforces strict Content Security Policies (CSP).
  - **CORS**: Strictly tailored Cross-Origin Resource Sharing configs to prevent unauthorized domain access.
- **Authentication**: Clerk's Node SDK (`@clerk/clerk-sdk-node`) is natively integrated to verify incoming client JWTs cryptographically before attempting database mutations.

---

## 🏗️ Core API Infrastructure

### 1. User Synchronization (`/api/auth`)
Seamlessly listens to the frontend and acts as the source of truth for user states. When a user creates an account via Clerk, it syncs those credentials into the MongoDB database with a default `pending` status, restricting application access until reviewed.

### 2. Admin Moderation Controller (`/api/admin`)
Exclusively locked behind the `isAdmin` middleware. Exposes endpoints to iterate over the user base, analyze demographics, and mutate user access states (Approve / Reject / Revoke).

### 3. Stateful User Data (`/api/favorites` & `/api/history`)
End-to-end controllers managing individual user preferences. Protects routes natively so users can only ever query or mutate their *own* distinct watchlists and viewing histories.

### 4. TMDB Proxy (`/api/tmdb`)
A dedicated service route that accepts requests generated by the frontend and securely fetches data from The Movie Database. This completely hides the `TMDB_API_KEY` from public clients and successfully bypasses extreme ISP/DNS Regional blocks (common in restricted regions) by leveraging the server's network.

### 5. Email Notification System
Integrated directly into the admin moderation flow. When a user's status is toggled, an asynchronous background task is triggered to send a branded HTML email via **Nodemailer** and **Gmail SMTP**.

## ⚙️ Local Development Setup

### Prerequisites
Make sure you have Node.js (v18+) installed, alongside an active MongoDB connection string (Atlas or Local).

1. **Install Dependencies**:
```bash
npm install
```

2. **Environment Variables**:
Create a `.env` file at the root of the `backend/` directory:
```env
# Server Config
PORT=5000
NODE_ENV=development

# Database
#MONGO_URI=mongodb+srv://<username>:<password>@cluster...
MONGO_URI=your_mongodb_connection_string

# Clerk Auth integration
CLERK_SECRET_KEY=your_clerk_secret_key
CLERK_PUBLISHABLE_KEY=your_clerk_publishable_key

# TMDB API domain (used in Content Security Policy for metadata requests)
# Provided by default as it is used to fetch movie/TV metadata from TMDB.
TMDB_API_KEY=your_tmdb_api_key
TMDB_BASE_URL=https://api.themoviedb.org/3

# CORS Whitelist
FRONTEND_URL=http://localhost:3000

# SMTP Email Configuration
# Automatically sends emails to users when their access status
# (approved, rejected, or revoked) is updated by the admin.
GMAIL_USER=your_gmail_account
# Use a Google App Password (not your main password).
# Generate it from Google Account → Security → App Passwords (requires 2FA).
GMAIL_APP_PASSWORD=your_gmail_app_password

# External Service URLs (Content Security Policy)
# These are placeholders for external streaming/embedding services.
# They are intentionally disabled and non-functional in this project.
VIDKING_BASE_URL=disabled
VIDKING_WILDCARD_URL=disabled
VIDKING_EMBED_URL=disabled
```

3. **Start the Development Server**:
```bash
npm run dev
# OR for standard execution
node server.js
```
The backend initializes gracefully on `:5000` (or your defined `$PORT`), boots the MongoDB pool via Mongoose, and logs a strict Health Check to confirm availability.

## 📧 Email Notification Setup (Gmail SMTP)

To enable automatic email notifications for approval/rejection:

1. **Enable 2FA**: Ensure 2-Step Verification is enabled on your Google Account.
2. **Generate App Password**:
   - Go to Google Account → Security → 2-Step Verification.
   - Scroll to the bottom and select **App Passwords**.
   - Select "Other (Custom name)" and enter `CineTv+`.
   - Copy the 16-character password provided.
3. **Configure .env**:
   - Set `GMAIL_USER` to your email.
   - Set `GMAIL_APP_PASSWORD` to the 16-character code (without spaces).
4. **Verification**: Use the admin panel to toggle a user's status; check the backend logs for `📧 Background email result`.

## 📦 Deployment (Render)

This Node.js server is fully optimized to be deployed as a Web Service on platforms like **Render** or **Railway**.

### Important Production Notes:
- **Environment**: Render securely injects `process.env.PORT`. Express embraces it automatically.
- **CORS Restriction**: Be totally sure to specify your exact Vercel deployed frontend URL in the `FRONTEND_URL` environment variable within your Render dashboard. The `server.js` file relies on it alongside an explicit whitelist to reject unauthorized API requests natively.
- **Database IP Permissions**: Ensure your MongoDB Atlas Network Access configuration allows traffic from `0.0.0.0/0` (Anywhere), as modern cloud providers dynamically shift container IPs!
